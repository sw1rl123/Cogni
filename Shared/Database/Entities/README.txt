Здесь лежат сущности, которые используются при передачи данных в бд или при получении записей из бд.
Бизнес-логика НЕ должна быть реализована с помощью этоих классов. Для работы с данными, полученными из БД, нужно использовать классы из папки Models. Дело в том, что классы-сущности связаны с контекстом данных, и, если изменить данные экземпляра класса и применить изменения, то данные в БД тоже изменяться.

!!!Если вам требуется создать конструктор в классе-сущности убедитесь, что там объявлен явный конструктор без параметров, если его нет - создайте. Это нужно для корректной работы EF!!!
